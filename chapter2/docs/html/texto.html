Expressões e declarações

No Capítulo 1 , criamos valores e aplicamos operadores a eles para obter novos valores. Criar valores como esse é a substância principal de qualquer programa JavaScript. Mas essa substância tem de ser enquadrada numa estrutura maior para ser útil. É isso que abordaremos neste capítulo.

Um fragmento de código que produz um valor é chamado de expressão . Todo valor escrito literalmente (como 22ou "psychoanalysis") é uma expressão. Uma expressão entre parênteses também é uma expressão, assim como um operador binário aplicado a duas expressões ou um operador unário aplicado a uma.

Isso mostra parte da beleza de uma interface baseada em linguagem. As expressões podem conter outras expressões de maneira semelhante à forma como as subsentenças nas línguas humanas são aninhadas – uma subsentença pode conter suas próprias subsentenças e assim por diante. Isso nos permite construir expressões que descrevem cálculos arbitrariamente complexos.

Se uma expressão corresponder a um fragmento de frase, uma instrução JavaScript corresponderá a uma frase completa. Um programa é uma lista de instruções.

O tipo mais simples de declaração é uma expressão com ponto e vírgula depois dela. Este é um programa:

1 ;
!falso;

É um programa inútil, no entanto. Uma expressão pode se contentar em produzir apenas um valor, que pode então ser usado pelo código anexo. No entanto, uma declaração é independente, por isso, se não afecta o mundo, é inútil. Ele pode exibir algo na tela, como acontece com console.log, ou alterar o estado da máquina de uma forma que afetará as instruções que virão depois dela. Essas alterações são chamadas de efeitos colaterais . As instruções do exemplo anterior apenas produzem os valores 1e trueimediatamente os descartam. Isso não deixa nenhuma impressão no mundo. Quando você executa este programa, nada observável acontece.

Em alguns casos, o JavaScript permite omitir o ponto e vírgula no final de uma instrução. Em outros casos, tem que estar lá, ou a próxima linha será tratada como parte da mesma instrução. As regras para quando ele pode ser omitido com segurança são um tanto complexas e sujeitas a erros. Portanto, neste livro, toda afirmação que precisa de ponto e vírgula sempre terá um. Recomendo que você faça o mesmo, pelo menos até aprender mais sobre as sutilezas da falta de ponto-e-vírgula.

Ligações

Como um programa mantém um estado interno? Como ele se lembra das coisas? Vimos como produzir novos valores a partir de valores antigos, mas isso não altera os valores antigos e o novo valor deve ser usado imediatamente ou se dissipará novamente. Para capturar e manter valores, o JavaScript fornece algo chamado vinculação ou variável :

vamos  pegar = 5 * 5 ;

Isso nos dá um segundo tipo de declaração. A palavra especial ( palavra-chave ) letindica que esta frase definirá uma ligação. É seguido pelo nome da ligação e, se quisermos atribuir-lhe imediatamente um valor, por um =operador e uma expressão.

O exemplo cria uma ligação chamada caughte a utiliza para capturar o número produzido pela multiplicação de 5 por 5.

Depois que uma ligação for definida, seu nome poderá ser usado como uma expressão. O valor dessa expressão é o valor que a ligação possui atualmente. Aqui está um exemplo:

seja  dez = 10 ;
console.log(dez * dez);
// → 100

Quando uma ligação aponta para um valor, isso não significa que ela esteja vinculada a esse valor para sempre. O =operador pode ser usado a qualquer momento em ligações existentes para desconectá-las de seu valor atual e fazer com que apontem para um novo:

deixe  humor = "luz" ;
console.log(humor);
// → 
humor claro = "escuro" ;
console.log(humor);
// → escuro

Você deve imaginar as amarrações como tentáculos em vez de caixas. Eles não contêm valores; eles os apreendem – duas ligações podem referir-se ao mesmo valor. Um programa pode acessar apenas os valores aos quais ainda possui uma referência. Quando você precisa se lembrar de algo, você cria um tentáculo para segurá-lo ou recoloca um de seus tentáculos existentes nele.

Vejamos outro exemplo. Para lembrar a quantidade de dólares que Luigi ainda lhe deve, você cria uma vinculação. Quando ele devolve $ 35, você atribui um novo valor a esta vinculação:

deixe  luigisDebt = 140 ;
luigisDívida = luigisDívida - 35 ;
console.log(luigisDebt);
// → 105

Quando você define uma ligação sem lhe dar um valor, o tentáculo não tem nada para agarrar, então termina no ar. Se você solicitar o valor de uma ligação vazia, obterá o valor undefined.

Uma única letinstrução pode definir múltiplas ligações. As definições devem ser separadas por vírgulas:

seja  um = 1 , dois = 2 ;
console.log(um + dois);
// → 3

As palavras vare consttambém podem ser usadas para criar ligações, de forma semelhante a let:

var  nome = "Ayda" ;
 const  saudação = "Olá" ;
console.log(saudação + nome);
// → Olá Ayda

A primeira delas, var(abreviação de “variável”), é a forma como as ligações eram declaradas no JavaScript anterior a 2015, quando letainda não existiam. Voltarei à maneira precisa como isso difere letno próximo capítulo . Por enquanto, lembre-se de que ele faz basicamente a mesma coisa, mas raramente o usaremos neste livro porque ele se comporta de maneira estranha em algumas situações.

A palavra constsignifica constante . Ele define uma ligação constante, que aponta para o mesmo valor enquanto durar. Isso é útil para ligações que apenas dão um nome a um valor para que você possa consultá-lo facilmente mais tarde.

Nomes de ligação

Os nomes de ligação podem ser qualquer sequência de uma ou mais letras. Os dígitos podem fazer parte de nomes vinculativos — catch22é um nome válido, por exemplo — mas o nome não deve começar com um dígito. Um nome vinculativo pode incluir cifrões ( $) ou sublinhados ( _), mas nenhuma outra pontuação ou caracteres especiais.

Palavras com significado especial, como let, são palavras-chave e não podem ser usadas como nomes vinculativos. Há também uma série de palavras “reservadas para uso” em versões futuras do JavaScript, que também não podem ser usadas como nomes vinculativos. A lista completa de palavras-chave e palavras reservadas é bastante longa:

break case catch class const continue depurador padrão
delete do else enum export estende false finalmente para
função if implementa interface de importação em instanceof let
novo pacote privado protegido público retorno estático super
mude este lance para verdadeiro, tente typeof var void enquanto estiver com rendimento

Não se preocupe em memorizar esta lista. Quando a criação de uma associação produz um erro de sintaxe inesperado, verifique se você está tentando definir uma palavra reservada.

O ambiente

A coleção de ligações e seus valores que existem em um determinado momento é chamada de ambiente . Quando um programa é iniciado, este ambiente não está vazio. Ele sempre contém ligações que fazem parte do padrão da linguagem e, na maioria das vezes, também possui ligações que fornecem maneiras de interagir com o sistema circundante. Por exemplo, em um navegador, existem funções para interagir com o site atualmente carregado e para ler entradas do mouse e do teclado.

Funções

Muitos dos valores fornecidos no ambiente padrão possuem o tipo function . Uma função é um pedaço de programa envolvido em um valor. Esses valores podem ser aplicados para executar o programa empacotado. Por exemplo, em um ambiente de navegador, a ligação promptcontém uma função que mostra uma pequena caixa de diálogo solicitando a entrada do usuário. É usado assim:

prompt( "Digite a senha" );

A execução de uma função é chamada de invocação , chamada ou aplicação . Você pode chamar uma função colocando parênteses após uma expressão que produz um valor de função. Normalmente você usará diretamente o nome da ligação que contém a função. Os valores entre parênteses são fornecidos ao programa dentro da função. No exemplo, a promptfunção usa a string que fornecemos como texto a ser mostrado na caixa de diálogo. Os valores dados às funções são chamados de argumentos . Funções diferentes podem precisar de um número diferente ou de tipos diferentes de argumentos.

A promptfunção não é muito usada na programação web moderna, principalmente porque você não tem controle sobre a aparência da caixa de diálogo resultante, mas pode ser útil em programas e experimentos de brinquedo.

A função console.log

Nos exemplos, usei console.logpara gerar valores. A maioria dos sistemas JavaScript (incluindo todos os navegadores modernos e Node.js) fornece uma console.logfunção que grava seus argumentos em algum dispositivo de saída de texto. Nos navegadores, a saída chega ao console JavaScript. Esta parte da interface do navegador fica oculta por padrão, mas a maioria dos navegadores a abre quando você pressiona F12 ou, em um Mac, comando - opção -I. Se isso não funcionar, pesquise nos menus um item chamado Ferramentas do desenvolvedor ou similar.

Ao executar os exemplos (ou seu próprio código) nas páginas deste livro, console.loga saída será mostrada após o exemplo, em vez de no console JavaScript do navegador.

seja  x = 30 ;
console.log( "o valor de x é" , x);
 // → o valor de x é 30

Embora os nomes de ligação não possam conter caracteres de ponto final, console.logeles possuem um. Isso ocorre porque console.lognão é uma ligação simples, mas uma expressão que recupera a logpropriedade do valor mantido pela consoleligação. Valores de retorno

Mostrar uma caixa de diálogo ou escrever texto na tela é um efeito colateral . Muitas funções são úteis devido aos efeitos colaterais que produzem. As funções também podem produzir valores e, nesse caso, não precisam ter um efeito colateral para serem úteis. Por exemplo, a função Math.maxrecebe qualquer quantidade de argumentos numéricos e retorna o maior:

console.log(Math.max( 2,4 ) ) ;
 // → 4

Quando uma função produz um valor, diz-se que ela retorna esse valor. Qualquer coisa que produza um valor é uma expressão em JavaScript, o que significa que chamadas de função podem ser usadas em expressões maiores. No código a seguir, uma chamada para Math.min, que é o oposto de Math.max, é usada como parte de uma expressão positiva:

console.log(Math.min( 2 , 4 ) + 100 );
 // → 102

Controle de fluxo

Quando o seu programa contém mais de uma instrução, as instruções são executadas como se fossem uma história, de cima para baixo. Por exemplo, o programa a seguir possui duas instruções. A primeira pede um número ao usuário, e a segunda, que é executada após a primeira, mostra o quadrado desse número:

deixe  theNumber = Number(prompt( "Escolha um número" ));
console.log( "Seu número é a raiz quadrada de " +
            oNúmero * oNúmero);

A função Numberconverte um valor em um número. Precisamos dessa conversão porque o resultado de prompté um valor de string e queremos um número. Existem funções semelhantes chamadas Stringe Booleanque convertem valores para esses tipos.

Execução condicional

Nem todos os programas são caminhos retos. Podemos, por exemplo, querer criar um ramal onde o programa tome o ramal adequado com base na situação em questão. Isso é chamado de execução condicional .

A execução condicional é criada com a ifpalavra-chave em JavaScript. No caso simples, queremos que algum código seja executado se, e somente se, uma determinada condição for válida. Poderíamos, por exemplo, querer mostrar o quadrado da entrada apenas se a entrada for realmente um número:

deixe  theNumber = Number(prompt( "Escolha um número" ));
 if (!Number.isNaN(oNúmero)) {
  console.log( "Seu número é a raiz quadrada de " +
              oNúmero * oNúmero);
}

Com esta modificação, se você inserir “papagaio”, nenhuma saída será mostrada.

A ifpalavra-chave executa ou ignora uma instrução dependendo do valor de uma expressão booleana. A expressão decisiva é escrita após a palavra-chave, entre parênteses, seguida da instrução a ser executada.

A Number.isNaNfunção é uma função JavaScript padrão que retorna trueapenas se o argumento fornecido for NaN. A Numberfunção retorna NaNquando você fornece uma string que não representa um número válido. Assim, a condição se traduz em “a menos que theNumbernão seja um número, faça isso”.

A instrução após the ifestá entre colchetes ( {e }) neste exemplo. As chaves podem ser usadas para agrupar qualquer número de instruções em uma única instrução, chamada bloco . Você também poderia tê-los omitido neste caso, já que eles contêm apenas uma única instrução, mas para evitar ter que pensar se eles são necessários, a maioria dos programadores JavaScript os utiliza em todas as instruções agrupadas como esta. Seguiremos principalmente essa convenção neste livro, exceto por algumas frases ocasionais.

if ( 1 + 1 == 2 ) console.log( "É verdade" );
 // → É verdade

Freqüentemente, você não terá apenas um código que é executado quando uma condição é verdadeira, mas também um código que trata do outro caso. Este caminho alternativo é representado pela segunda seta no diagrama. Você pode usar a elsepalavra-chave, junto com if, para criar dois caminhos de execução alternativos separados:

deixe  theNumber = Number(prompt( "Escolha um número" ));
 if (!Number.isNaN(oNúmero)) {
  console.log( "Seu número é a raiz quadrada de " +
              oNúmero * oNúmero);
} outro {
  console.log( "Ei. Por que você não me deu um número?" );
}

Se você tiver mais de dois caminhos para escolher, poderá “encadear” vários if/ elsepares. Aqui está um exemplo:

deixe  num = Número(prompt( "Escolha um número" ));

se (num < 10 ) {
  console.log( "Pequeno" );
} senão  if (num < 100 ) {
  console.log( "Médio" );
} outro {
  console.log( "Grande" );
}

O programa primeiro verificará se numé menor que 10. Se for, ele escolhe esse ramo, mostra "Small"e pronto. Se não for, ele pega o elsebranch, que contém um segundo if. Se a segunda condição ( < 100) for válida, isso significa que o número é pelo menos 10, mas inferior a 100, e "Medium"é mostrado. Caso contrário, o segundo e último elseramo é escolhido.

while e fazer loops

Considere um programa que produz todos os números pares de 0 a 12. Uma maneira de escrever isso é a seguinte:

console.log( 0 );
console.log( 2 );
console.log( 4 );
console.log( 6 );
console.log( 8 );
console.log( 10 );
console.log( 12 );

Isso funciona, mas a ideia de escrever um programa é fazer com que algo funcione menos , e não mais. Se precisássemos de todos os números pares inferiores a 1.000, esta abordagem seria impraticável. O que precisamos é de uma maneira de executar um trecho de código várias vezes. Esta forma de fluxo de controle é chamada de loop .

O fluxo de controle em loop nos permite voltar a algum ponto do programa onde estávamos antes e repeti-lo com o estado atual do programa. Se combinarmos isso com uma ligação que conte, podemos fazer algo assim:

deixe  número = 0 ;
 enquanto (número <= 12 ) {
  console.log(número);
  número = número + 2 ;
}
// → 0 
// → 2 
//… etc.

Uma instrução começando com a palavra-chave whilecria um loop. A palavra whileé seguida por uma expressão entre parênteses e depois por uma declaração, muito parecida com if. O loop continua entrando nessa instrução enquanto a expressão produz um valor que é fornecido truequando convertida para booleano.

A numbervinculação demonstra como uma vinculação pode acompanhar o progresso de um programa. Cada vez que o loop se repete, numberobtém um valor 2 a mais que seu valor anterior. No início de cada repetição, é comparado com o número 12 para decidir se o trabalho do programa está concluído.

Como um exemplo que realmente faz algo útil, podemos agora escrever um programa que calcula e mostra o valor de 2 10 (2 elevado à décima potência). Usamos duas ligações: uma para acompanhar nosso resultado e outra para contar quantas vezes multiplicamos esse resultado por 2. O loop testa se a segunda ligação já atingiu 10 e, se não, atualiza ambas as ligações.

deixe  resultado = 1 ;
 deixe  contador = 0 ;
 enquanto (contador < 10 ) {
  resultado = resultado * 2 ;
  contador = contador + 1 ;
}
console.log(resultado);
// → 1024

O contador também poderia ter iniciado 1e verificado <= 10, mas por razões que ficarão aparentes no Capítulo 4 , é uma boa ideia se acostumar a contar a partir de 0.

Observe que o JavaScript também possui um operador para exponenciação ( 2 ** 10), que você usaria para calcular isso em código real — mas isso teria arruinado o exemplo.

Um doloop é uma estrutura de controle semelhante a um whileloop. Ele difere apenas em um ponto: um doloop sempre executa seu corpo pelo menos uma vez e começa a testar se deve parar somente após a primeira execução. Para refletir isso, o teste aparece após o corpo do loop:

deixe  seuNome ;
 fazer {
  seuNome = prompt( "Quem é você?" );
} while (!seuNome);
console.log( "Olá" + seuNome);

Este programa forçará você a inserir um nome. Ele perguntará repetidamente até obter algo que não seja uma string vazia. A aplicação do !operador converterá um valor para o tipo booleano antes de negá-lo, e todas as strings, exceto ""serão convertidas para true. Isso significa que o loop continua até que você forneça um nome não vazio.

Código de recuo

Nos exemplos, adicionei espaços na frente de declarações que fazem parte de alguma declaração maior. Esses espaços não são necessários – o computador aceitará o programa perfeitamente sem eles. Na verdade, até mesmo as quebras de linha nos programas são opcionais. Você poderia escrever um programa como uma única linha longa, se quisesse.

O papel desse recuo dentro dos blocos é fazer com que a estrutura do código se destaque para os leitores humanos. No código onde novos blocos são abertos dentro de outros blocos, pode ser difícil ver onde um bloco termina e outro começa. Com a indentação adequada, a forma visual de um programa corresponde à forma dos blocos dentro dele. Gosto de usar dois espaços para cada bloco aberto, mas os gostos diferem – algumas pessoas usam quatro espaços e outras usam caracteres de tabulação. O importante é que cada novo bloco acrescente a mesma quantidade de espaço.

if (falso! = verdadeiro) {
  console.log( "Isso faz sentido." );
   se ( 1 < 2 ) {
    console.log( "Nenhuma surpresa." );
  }
}

A maioria dos programas editores de código (incluindo o deste livro) ajudará, recuando automaticamente as novas linhas na quantidade adequada.

para laços

Muitos loops seguem o padrão mostrado nos whileexemplos. Primeiro, uma ligação de “contador” é criada para rastrear o progresso do loop. Depois vem um whileloop, geralmente com uma expressão de teste que verifica se o contador atingiu seu valor final. No final do corpo do loop, o contador é atualizado para acompanhar o progresso.

Como esse padrão é tão comum, o JavaScript e linguagens similares fornecem uma forma um pouco mais curta e abrangente, o forloop:

for ( seja  número = 0 ; número <= 12 ; número = número + 2 ) {
  console.log(número);
}
// → 0 
// → 2 
//… etc.

Este programa é exatamente equivalente ao exemplo anterior de impressão de números pares. A única mudança é que todas as instruções relacionadas ao “estado” do loop são agrupadas após for.

Os parênteses após uma forpalavra-chave devem conter dois pontos e vírgulas. A parte antes do primeiro ponto e vírgula inicializa o loop, geralmente definindo uma ligação. A segunda parte é a expressão que verifica se o loop deve continuar. A parte final atualiza o estado do loop após cada iteração. Na maioria dos casos, isso é mais curto e claro do que uma whileconstrução.

Saindo de um loop

Fazer com que a condição de loop seja produzida falsenão é a única maneira de um loop terminar. A breakinstrução tem o efeito de saltar imediatamente para fora do loop envolvente. Seu uso é demonstrado no programa a seguir, que encontra o primeiro número maior ou igual a 20 e divisível por 7:

for ( deixe  atual = 20 ; ; atual = atual + 1 ) {
   if (atual% 7 == 0 ) {
    console.log(atual);
    quebrar ;
  }
}
// → 21

Usar o operador resto ( %) é uma maneira fácil de testar se um número é divisível por outro número. Se for, o resto da divisão é zero.

A forconstrução no exemplo não possui uma parte que verifica o final do loop. Isso significa que o loop nunca irá parar a menos que a breakinstrução interna seja executada.

Se você removesse essa breakinstrução ou escrevesse acidentalmente uma condição final que sempre produz true, seu programa ficaria preso em um loop infinito . Um programa preso em um loop infinito nunca terminará a execução, o que geralmente é uma coisa ruim.

Se você criar um loop infinito em um dos exemplos destas páginas, normalmente será perguntado se deseja interromper o script após alguns segundos. Se isso falhar, você terá que fechar a guia em que está trabalhando para recuperar.

A continuepalavra-chave é semelhante breakporque influencia o progresso de um loop. Quando continueé encontrado em um corpo de loop, o controle salta para fora do corpo e continua com a próxima iteração do loop.

Atualizando vinculações de forma sucinta

Especialmente durante o loop, um programa muitas vezes precisa “atualizar” uma ligação para manter um valor baseado no valor anterior dessa ligação.

contador = contador + 1 ;

JavaScript fornece um atalho para isso:

contador += 1 ;

Atalhos semelhantes funcionam para muitos outros operadores, como result *= 2duplicar resultou counter -= 1contar regressivamente.

Isso nos permite encurtar ainda mais nosso exemplo de contagem:

for ( seja  número = 0 ; número <= 12 ; número += 2 ) {
  console.log(número);
}

Para counter += 1e counter -= 1, existem equivalentes ainda mais curtos: counter++e counter--.

Despachando um valor com switch

Não é incomum que o código fique assim:

if (x == "valor1" ) ação1();
 senão  if (x == "valor2" ) action2();
 senão  if (x == "valor3" ) action3();
 senão defaultAction();

Existe uma construção chamada switchque pretende expressar tal “despacho” de forma mais direta. Infelizmente, a sintaxe que o JavaScript usa para isso (que herdou da linha de linguagens de programação C/Java) é um tanto estranha – uma cadeia de ifinstruções pode parecer melhor. Aqui está um exemplo:

switch (prompt( "Como está o tempo?" )) {
   case  "chuvoso" :
    console.log( "Lembre-se de trazer um guarda-chuva." );
     quebrar ;
   caso  "ensolarado" :
    console.log( "Vista-se levemente." );
   caso  "nublado" :
    console.log( "Vá para fora." );
     quebrar ;
   padrão :
    console.log( "Tipo de clima desconhecido!" );
     quebrar ;
}

Você pode colocar qualquer número de caserótulos dentro do bloco aberto por switch. O programa começará a ser executado no rótulo que corresponde ao valor fornecido switchou em defaultse nenhum valor correspondente for encontrado. Ele continuará em execução, mesmo em outros rótulos, até chegar a uma breakinstrução. Em alguns casos, como "sunny"no exemplo, isso pode ser usado para compartilhar algum código entre os casos (é recomendável sair de casa tanto em dias ensolarados quanto nublados). Porém, tenha cuidado - é fácil esquecer tal break, o que fará com que o programa execute código que você não deseja que seja executado.

Capitalização

Os nomes das ligações podem não conter espaços, mas muitas vezes é útil usar várias palavras para descrever claramente o que a ligação representa. Estas são basicamente suas escolhas para escrever um nome vinculativo com várias palavras:

pequenatartaruga fuzzy
fuzzy_pequena_tartaruga
FuzzyLittleTurtle
fuzzyLittleTurtle

O primeiro estilo pode ser difícil de ler. Gosto bastante da aparência dos sublinhados, embora esse estilo seja um pouco doloroso de digitar. As funções JavaScript padrão, e a maioria dos programadores de JavaScript, seguem o estilo final – elas colocam todas as palavras em maiúscula, exceto a primeira. Não é difícil se acostumar com pequenas coisas como essa, e códigos com estilos de nomenclatura mistos podem ser difíceis de ler, por isso seguimos esta convenção.

Em alguns casos, como na Numberfunção, a primeira letra de uma ligação também é maiúscula. Isso foi feito para marcar esta função como um construtor. Ficará claro o que é um construtor no Capítulo 6 . Por enquanto, o importante é não se incomodar com esta aparente falta de consistência.

Comentários

Freqüentemente, o código bruto não transmite todas as informações que você deseja que um programa transmita aos leitores humanos, ou as transmite de uma forma tão enigmática que as pessoas podem não entendê-las. Outras vezes, você pode querer incluir apenas algumas idéias relacionadas como parte de seu programa. É para isso que servem os comentários .

Um comentário é um pedaço de texto que faz parte de um programa, mas é completamente ignorado pelo computador. JavaScript tem duas maneiras de escrever comentários. Para escrever um comentário de linha única, você pode usar duas barras ( //) e depois o texto do comentário:

deixe  contaSaldo = calcularSaldo(conta);
 //É um buraco verde onde canta um rio
contaBalance.adjust();
// Pegando farrapos brancos na grama loucamente. 
deixe  relatório = novo Relatório();
 // Onde o sol ressoa na orgulhosa montanha:
addToReport(saldoconta, relatório);
// É um pequeno vale, espumando como luz num copo.

Um //comentário vai apenas até o final da linha. Uma seção de texto entre /*e */será totalmente ignorada, independentemente de conter quebras de linha. Isso é útil para adicionar blocos de informações sobre um arquivo ou parte do programa:

  números de telefone e números de série de produtos que comprei 
  . Obviamente gosta de mim, então decidi mantê-lo. 
*/

Resumo

Agora você sabe que um programa é construído a partir de instruções, que às vezes contêm mais instruções. As declarações tendem a conter expressões, que podem ser construídas a partir de expressões menores.

Colocar instruções uma após a outra fornece um programa que é executado de cima para baixo. Você pode introduzir distúrbios no fluxo de controle usando instruções condicionais ( if, else, and switch) e de loop ( while, do, and for).

As ligações podem ser usadas para arquivar dados sob um nome e são úteis para rastrear o estado do seu programa. O ambiente é o conjunto de ligações definidas. Os sistemas JavaScript sempre colocam uma série de ligações padrão úteis em seu ambiente.

Funções são valores especiais que encapsulam uma parte do programa. Você pode invocá-los escrevendo functionName(argument1, argument2). Tal chamada de função é uma expressão e pode produzir um valor.

Para trabalhar com duas dimensões, você precisará de um loop dentro de outro loop.
